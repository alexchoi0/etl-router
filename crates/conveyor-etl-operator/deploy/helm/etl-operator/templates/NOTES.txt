ETL Operator has been installed!

1. Verify the operator is running:
   kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "conveyor-etl-operator.name" . }}

2. Check CRDs are installed:
   kubectl get crd | grep conveyor.etl

3. Create an ETL Router cluster:
   cat <<EOF | kubectl apply -f -
   apiVersion: conveyor.etl/v1
   kind: EtlRouterCluster
   metadata:
     name: my-cluster
     namespace: default
   spec:
     replicas: 3
     image: etl-router:latest
     storage:
       storageClass: standard
       size: 10Gi
   EOF

4. Monitor cluster status:
   kubectl get etlrouterclusters -w
   kubectl get erc  # shortname

5. Create a pipeline:
   cat <<EOF | kubectl apply -f -
   apiVersion: conveyor.etl/v1
   kind: Source
   metadata:
     name: my-source
   spec:
     grpc:
       endpoint: source-svc:50051
   ---
   apiVersion: conveyor.etl/v1
   kind: Sink
   metadata:
     name: my-sink
   spec:
     grpc:
       endpoint: sink-svc:50051
   ---
   apiVersion: conveyor.etl/v1
   kind: Pipeline
   metadata:
     name: my-pipeline
   spec:
     source: my-source
     sink: my-sink
     enabled: true
   EOF

6. View resources:
   kubectl get sources,transforms,sinks,pipelines

For more information, visit the ETL Router documentation.
